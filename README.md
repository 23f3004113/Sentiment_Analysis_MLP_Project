## Comment Sentiment Analysis

This repository contains a robust machine learning pipeline developed for a Kaggle competition focused on predicting the sentiment categories of public comments. The solution successfully classifies high-dimensional text data and metadata while strictly adhering to project constraints that prohibited the use of external NLP libraries like NLTK.

🚀 Project Overview

The objective was to accurately predict the sentiment label of comments based on textual content and associated features such as upvotes, downvotes, and emoticons.

Key Highlights:

Ensemble Modeling: Leveraged a VotingClassifier to combine multiple linear models, optimizing performance on sparse text data.

Data Integrity: Executed the train-test split prior to TF-IDF vectorization to eliminate data leakage. Validation indices were strictly established before attempting to slice data matrices, ensuring a reliable evaluation framework.

Library Constraints: Handled all text cleaning, tokenization, and vectorization using only core pandas and scikit-learn functionality.

🛠️ Tech Stack

Language: Python

Data Processing & EDA: Pandas, NumPy, Matplotlib, Seaborn

Feature Engineering: Scikit-Learn (TfidfVectorizer, RobustScaler)

Machine Learning: Scikit-Learn (LogisticRegression, LinearSVC, SGDClassifier, VotingClassifier)

📊 Pipeline Architecture
1. Exploratory Data Analysis (EDA)

Investigated target label distribution, identifying and addressing class imbalances.

Analyzed correlations between comment length and engagement metrics (upvotes/downvotes).

Visualized distributions to guide feature scaling decisions.

2. Preprocessing & Feature Engineering

Text Cleaning: Handled missing values and normalized text completely within pandas.

Numerical Scaling: Applied RobustScaler to metadata features to mitigate the impact of extreme outliers in upvotes and downvotes.

Vectorization: Transformed raw text into numerical matrices using TfidfVectorizer, carefully fitting the vectorizer exclusively on the training set to prevent information bleed into the validation set.

3. Model Architecture

To achieve high predictive accuracy, a Voting Classifier was implemented to ensemble three distinct linear models:

Logistic Regression: Provided a strong, interpretable probabilistic baseline.

Linear SVC (Support Vector Classifier): Highly effective at finding optimal hyperplanes in the high-dimensional sparse matrices generated by TF-IDF.

SGD Classifier: Efficiently optimized the loss function using stochastic gradient descent.

By combining these models, the final classifier benefited from reduced variance and improved generalization on the test data.
